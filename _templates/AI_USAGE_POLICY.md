## 1. 使用申請フロー

### 対象

Claude Code（AI コーディング支援ツール）および接続する MCP サーバー。

### 申請・承認プロセス

```
1. 利用申請: Google Form / 社内チケットで申請
   - 用途・対象プロジェクト・取り扱いデータの種類を記載

2. 審査（情報システム部 or セキュリティ担当）:
   - 取り扱いデータに個人情報・機密情報が含まれないかチェック
   - MCPサーバーは社内ホワイトリストに掲載済みのもののみ許可

3. 承認後: Managed Settings（組織ポリシー）が自動適用された端末で利用開始

4. 定期レビュー: 四半期ごとに利用状況・ログをレビュー
```

### MCP サーバー追加申請

新しい MCP サーバーの追加は必ず申請・審査を経ること。
`enableAllProjectMcpServers: false` により自動有効化は無効化済み（CVE-2025-59536対応）。

## 2. 入力禁止情報

以下の情報は Claude Code のプロンプト・コンテキストに**絶対に含めない**。

### 禁止情報一覧

| 種別 | 具体例 | 根拠 |
|------|-------|------|
| **採用候補者データ** | 適性診断スコア、評価シート、面接メモ、氏名・連絡先 | 改正個人情報保護法（要配慮個人情報の第三者提供規制） |
| **本番DB接続情報** | 本番環境の接続文字列、DB認証情報 | 情報漏洩・不正アクセスリスク |
| **未公開財務数値** | 売上予測、投資家向け資料、取締役会議事録 | 内部者取引規制・IPO審査要件 |
| **顧客個人情報** | 氏名・住所・電話番号・メールアドレスを含むデータ | 個人情報保護法 第23条（第三者提供の制限） |
| **従業員情報** | 給与データ、人事評価、健康診断結果 | 要配慮個人情報・従業員プライバシー |
| **認証情報** | API キー、パスワード、秘密鍵（`.env` 等） | 機密情報保護・不正アクセス防止 |

### 代替手段（マスキング）

実際のデータが必要な場合は、ダミーデータ・マスキングデータに置き換えて入力する。

```
悪い例: 田中 太郎（34歳）の適性診断スコアは78点で...
良い例: 候補者Aの適性診断スコアは[数値]で...

悪い例: DATABASE_URL=postgresql://admin:prod-password@prod-db.example.com/myapp
良い例: DATABASE_URL=postgresql://user:password@localhost:5432/myapp_dev
```

## 3. ログ保存方針（J-SOX 対応）

### 記録内容

Claude Code による全ツール実行が `.context/tool-log.jsonl` に自動記録される。

```json
{
  "timestamp": "2026-03-21T10:00:00.000Z",
  "session_id": "session-abc123",
  "operator": "keiji",
  "project": "my-project",
  "tool": "Edit",
  "input_summary": "src/billing/invoice.ts",
  "success": true
}
```

### 保存期間・方法

- **保存期間**: 7年（日本版SOX法の財務報告関連記録の要件に準拠）
- **保存方法**: 月次でクラウドストレージ（S3等）にアーカイブ（暗号化・バージョニング有効）
- **アクセス制限**: 情報システム部・監査担当のみが閲覧可能

### 改ざん防止

- ログファイルへの書き込みは append-only
- クラウドストレージの Object Lock（WORM）を活用
- ログ改ざん・削除には承認プロセスが必要

## 4. レビュー義務

### 開発者の義務

1. **コードレビュー**: AI が生成したコードは、人間によるコードレビューを必須とする
2. **セキュリティチェック**: `sentinel`（SAST）エージェントによる静的解析を CI/CD に組み込む
3. **アウトプット責任**: AI の出力物に対する最終責任は人間の開発者が負う

### チームリーダーの義務

1. **四半期レビュー**: ログから異常な利用（大量データ入力・禁止情報の入力試行等）がないか確認
2. **ポリシー周知**: 新メンバーへのオンボーディングでこのポリシーを説明
3. **インシデント報告**: 禁止情報の誤入力・情報漏洩が疑われる場合は即時セキュリティ担当へ報告

### 情報システム部の義務

1. **Managed Settings 管理**: 組織ポリシーの更新・配布
2. **MCPサーバーホワイトリスト**: 承認済みMCPサーバーリストの維持
3. **監査ログレビュー**: 四半期ごとのログ分析・異常検知レポート作成

## 5. 違反時の対応

| 違反内容 | 対応 |
|---------|------|
| 禁止情報の誤入力 | 即時報告 → インシデント記録 → Anthropic サポートへの問い合わせ（必要に応じ） |
| 承認なしの MCP サーバー追加 | アクセス権停止 → 調査 → 再教育 |
| ログの改ざん・削除 | 懲戒処分の対象 |
| ポリシー違反の繰り返し | AIツール利用権限の停止 |
