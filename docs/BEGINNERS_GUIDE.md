# AIって何ができるの？ ── 中学生のための Agent Orchestrator ガイド

> この文書は「ChatGPTで雑談したことはあるけど、それ以外は知らない」という人向けに書かれています。
> 専門用語はできるだけ使いません。出てきたときは必ず説明します。

---

## 1. まず「AI」って何だろう

あなたが普段使っている ChatGPT は、AIの一種です。

質問すると答えてくれる。相談に乗ってくれる。作文を手伝ってくれる。
でも、**AIにできることは「会話」だけじゃない**んです。

| あなたが知っているAI | 実はAIにできること |
|---|---|
| 質問に答える | プログラムを書く |
| 相談に乗る | ファイルを作る・編集する |
| 文章を考える | バグ（プログラムの間違い）を見つけて直す |
| 翻訳する | ウェブサイトを自動で操作する |
| | データを分析してグラフを作る |
| | チームを組んで大きな仕事を分担する |

右側の「実はできること」をやるためのAIが、このプロジェクトのテーマです。

---

## 2. 「開発」って何？

**開発（かいはつ）** = ソフトウェアを作ること。

あなたが毎日使っているもの──LINE、YouTube、Instagram、ゲーム。
これらは全部、誰かが **プログラム（コンピュータへの命令書）** を書いて作っています。

```
こんなイメージ:

  企画する → 設計する → コードを書く → テストする → 公開する
  「こんなアプリ    「どういう      「実際に       「ちゃんと      「みんなが
   作りたい！」    仕組みに       命令を        動くか       使えるように
               する？」     書く」       確認する」     する」
```

この「コードを書く」「テストする」という部分を、**AIに手伝ってもらう**のが最近の流れです。
手伝うどころか、AIがほぼ全部やってくれることもあります。

---

## 3. Claude Code って何？

**Claude Code（クロード・コード）** は、Anthropic という会社が作った「開発ができるAI」です。

普通のChatGPTとの違いを、学校の例えで説明します。

| | ChatGPT（普通のAI） | Claude Code（開発AI） |
|---|---|---|
| 例えると | 何でも答えてくれる先生 | 一緒に手を動かしてくれる先輩 |
| できること | 「教えて」→「答えるよ」 | 「これ作って」→「はい、作ったよ」 |
| ファイル操作 | できない | **ファイルを作る・読む・書き換えられる** |
| コマンド実行 | できない | **コンピュータに直接命令できる** |
| 会話の形 | ブラウザのチャット画面 | **ターミナル**（黒い画面）で動く |

つまり、Claude Code は **「話すだけ」じゃなくて「実際に作業してくれる」AI** です。

「ログイン画面を作って」と言えば、本当にファイルを作ってコードを書いてくれます。

---

## 4. でも、1人だと限界がある

Claude Code はすごいけど、1人（1つのAI）だと困ることがあります。

たとえば、文化祭の準備を想像してください。

> **あなた**: 「文化祭の出し物、全部やって！」
>
> **1人のAI**: 「えっ、看板作って、料理して、会計して、BGM選んで、
>              案内のチラシも作って......全部1人は無理です！」

人間と同じで、**AIも1人で全部やるのは大変**なんです。
特に：

- 作業が多すぎると、前にやったことを忘れる
- 得意なことと苦手なことがある
- 同時に複数のことはできない

じゃあどうする？

**→ チームを作ればいい。**

## 5. 「オーケストラ」という考え方

ここでやっと、このプロジェクトの名前「**Agent Orchestrator（エージェント・オーケストレーター）**」の話です。

### オーケストラって何？

音楽のオーケストラを思い浮かべてください。

```
              🎼 指揮者（しきしゃ）
              ／    |    ＼
         🎻        🎺        🥁
      バイオリン   トランペット    ドラム
       （弦楽器）   （管楽器）   （打楽器）
```

- **指揮者**は自分では楽器を弾かない。でも、誰がいつ演奏するかを決める
- **各パート**は自分の楽器を一生懸命弾く
- 全員がバラバラに弾いたら騒音。指揮者がまとめるから**美しい音楽**になる

このプロジェクトは、これと同じことを **AIで** やります。

### AIのオーケストラ

```
              🎼 Nexus（指揮者AI）
              ／    |    ＼
         🔍        🔨        🧪
        Scout     Builder     Radar
      （調査役）   （開発役）   （テスト役）
```

- **Nexus（ネクサス）** = 指揮者。自分ではコードを書かない。「誰に何をやらせるか」を決める
- **Scout（スカウト）** = 調査役。バグ（不具合）を見つける専門家
- **Builder（ビルダー）** = 開発役。実際にコードを書く職人
- **Radar（レーダー）** = テスト役。作ったものがちゃんと動くか確認する

他にも **68人のAI専門家** がいます（セキュリティ、デザイン、データ分析など）。

## 6. 実際にどう動くの？

たとえば「ログイン画面にバグがある」と報告があったとします。

### ステップ 1: 指揮者が聞く

```
あなた: 「ログイン画面でエラーが出る。直して」
Nexus:  「了解。まず調査が必要だな。Scout、調べてきて」
```

### ステップ 2: 調査役が調べる

```
Scout: 「原因わかりました。パスワードの文字数チェックにバグがあります。
        ファイルは login.js の42行目です」
Nexus: 「ありがとう。Builder、直して」
```

### ステップ 3: 開発役が直す

```
Builder: 「修正しました。login.js の42行目を書き換えました」
Nexus:   「Radar、テストして」
```

### ステップ 4: テスト役が確認

```
Radar: 「テスト全部通りました。問題ありません」
Nexus: 「完了。全部OKです」
```

**あなたがやったことは最初の一言だけ**。あとはAIチームが全部やってくれます。

## 8. なぜチームにするの？

1人のAIに全部やらせることもできます。でも、チームにする理由があります。

### 理由 1: 専門家は専門のことが上手い

バグを直すとき、「バグを探すのが得意なAI」と「コードを書くのが得意なAI」を分けたほうが、両方うまくいきます。

### 理由 2: 同時に複数のことができる

```
1人だと:  調査 → 開発 → テスト → レビュー （順番にやるから遅い）

チームだと: 調査（Scout）
           開発（Builder）  ← 同時にできる！
           テスト（Radar）
```

### 理由 3: ミスに気づける

Builder が書いたコードを Judge がチェックする。
人間でも「自分で書いた文章の間違いに気づけない」ことがありますよね。
別のAIがチェックすることで、ミスを減らせます。

### 理由 4: 安全に動ける

このシステムには **ガードレール（安全装置）** があります。

| レベル | 何が起きたら | どうする |
|--------|------------|---------|
| L1 | 小さな警告 | 記録だけして続行 |
| L2 | テストが少し失敗 | 自動で修正を試みる（最大3回） |
| L3 | テストが大量に失敗 | 元に戻して、やり直す |
| L4 | セキュリティ問題 | **即座に全停止** |

暴走しないように、ちゃんとブレーキがついています。

## 9. 「普通のAI」と「このシステム」の違い

まとめると、こういう違いがあります。

```
【普通のAI（ChatGPTなど）】

  あなた ←→ AI（1人）

  「〇〇って何？」「〇〇について教えて」
  → 答えが返ってくる
  → でも実際に何かを「作る」ことはない


【Agent Orchestrator】

  あなた → Nexus（指揮者）→ 専門家チーム（68人）→ 成果物
                                                    ↑
                                            実際にファイルが作られる
                                            コードが書かれる
                                            テストが実行される
                                            バグが修正される
```

**「教えてもらう」から「作ってもらう」へ。**

これが、このシステムの本質です。

## 10. もっと身近な例で

### 例 1: 学校のウェブサイトを作りたい

```
あなた: 「学校の文化祭のウェブサイトを作って」

→ Nexus が計画を立てる
→ Builder がHTMLとCSSを書く
→ Artisan がデザインを整える
→ Radar がスマホでも表示されるかテストする
→ 完成！
```

### 例 2: データを分析したい

```
あなた: 「この100件のアンケート結果をグラフにして」

→ Nexus が Analyst に任せる
→ Analyst がデータを分析する
→ Canvas がグラフを作る
→ 完成！
```

### 例 3: アプリのバグを直したい

```
あなた: 「ボタンを押しても反応しない」

→ Nexus が Scout に調査させる
→ Scout が原因を特定する
→ Builder が修正する
→ Radar がテストする
→ Sentinel がセキュリティを確認する
→ 完成！
```

## 11. これを使うには？

現時点では、このシステムを使うには以下が必要です。

1. **Claude Code** がインストールされたコンピュータ
2. **ターミナル**（黒い画面。コンピュータに文字で命令する場所）の基本操作
3. プログラミングの基本的な知識

「中学生にはまだ早い」と思うかもしれません。
でも、ここで大事なのは **こういう世界があると知ること** です。

AIは「おしゃべり相手」から「一緒に働くチームメイト」に進化しています。
あなたが大人になるころには、AIチームと一緒に仕事をするのが当たり前の世界になっているはずです。

## 12. 用語まとめ

最後に、この文書に出てきた言葉をまとめておきます。

| 用語 | 意味 |
|------|------|
| **AI（エーアイ）** | 人工知能。人間のように考えたり判断したりするコンピュータの仕組み |
| **プログラム** | コンピュータへの命令書。これを書くことを「プログラミング」という |
| **コード** | プログラムの中身（命令文）のこと |
| **バグ** | プログラムの間違い。意図しない動きをすること |
| **開発** | ソフトウェア（アプリやウェブサイト）を作ること |
| **ターミナル** | コンピュータに文字で命令する画面。黒い画面 |
| **Claude Code** | Anthropic社が作った、開発ができるAI |
| **エージェント** | 特定の役割を持ったAI。チームの一員 |
| **オーケストレーター** | 複数のエージェントをまとめる指揮者 |
| **リポジトリ** | プログラムのファイルをまとめて保管する場所 |
| **テスト** | 作ったものがちゃんと動くか確認すること |
| **セキュリティ** | 情報やシステムを守ること。不正アクセスを防ぐなど |
| **Git** | ファイルの変更履歴を記録する仕組み。「いつ誰が何を変えたか」がわかる |
